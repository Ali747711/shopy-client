# i18n — Korean Language Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English/Korean language toggle using `react-i18next` — no URL-based routing, no route restructuring.

**Architecture:** Client-side i18n via `react-i18next`. Translation JSON files organized by namespace. Language preference stored in `localStorage` and a cookie (for SSR reads). A `LanguageSwitcher` component in the dock nav. Server components receive translations via a lightweight utility that reads the JSON files directly.

**Tech Stack:** `i18next`, `react-i18next`, `i18next-browser-languagedetector`

- [ ] **Step 2: Create i18n settings**

Create `lib/i18n/settings.ts`:

```ts
export const defaultLocale = 'en'
export const locales = ['en', 'ko'] as const
export type Locale = (typeof locales)[number]

export const localeNames: Record<Locale, string> = {
  en: 'English',
  ko: '한국어',
}
```

- [ ] **Step 3: Create i18n client initializer**

Create `lib/i18n/index.ts`:

```ts
'use client'

import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'

import { defaultLocale, locales } from './settings'

import en from '@/messages/en.json'
import ko from '@/messages/ko.json'

if (!i18n.isInitialized) {
  i18n
    .use(LanguageDetector)
    .use(initReactI18next)
    .init({
      resources: { en: { translation: en }, ko: { translation: ko } },
      fallbackLng: defaultLocale,
      supportedLngs: [...locales],
      interpolation: { escapeValue: false },
      detection: {
        order: ['localStorage', 'navigator'],
        caches: ['localStorage'],
        lookupLocalStorage: 'shopy-lang',
      },
    })
}

export default i18n
```
